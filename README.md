# crypto-market-mcp

MCP-сервер с данными крипторынка и традиционных финансов. Не требует API-ключей —
использует бесплатные публичные эндпоинты CoinGecko, alternative.me и Yahoo Finance.

## Инструменты

| Инструмент | Что даёт |
|---|---|
| `get_global_market_data` | Суммарная капитализация рынка, объём 24ч, доминация BTC/ETH |
| `get_top_coins` | Топ монет по капитализации (цена, изменения 24ч/7д, объём) |
| `get_coin_details` | Подробности по одной монете (цена, ATH, supply, соцсети) |
| `search_coin` | Поиск CoinGecko id монеты по названию/тикеру |
| `get_trending_coins` | Что сейчас в трендах поиска на CoinGecko |
| `get_fear_greed_index` | Индекс страха и жадности крипторынка |
| `get_market_movers` | Топ растущих/падающих монет за 24ч |
| `get_traditional_markets` | S&P 500, Nasdaq, Dow, доходность 10Y, DXY, золото, нефть |

## Два варианта запуска

Сервер поддерживает два транспорта:

- **stdio** (`src/index.js`) — локальный процесс, только для Claude Code на этой же машине. Не виден в Cowork/claude.ai — они выполняют задачи в изолированной среде без доступа к локальным процессам.
- **HTTP** (`src/http.js`) — сервер, доступный по сети. После деплоя в облако его можно подключить как custom connector — тогда он будет виден **везде**: в Claude Code, Cowork и claude.ai.

Если нужен доступ только из Claude Code на своей машине — используй вариант A. Если нужен доступ из Cowork — обязательно нужен вариант B (деплой).

---

## Вариант A: локально, для Claude Code

### 1. Скачай и распакуй проект

Распакуй `crypto-market-mcp.zip` в удобную папку, например `~/mcp-servers/crypto-market-mcp`.

### 2. Установи зависимости

```bash
cd ~/mcp-servers/crypto-market-mcp
npm install
```

### 3. Добавь сервер в Claude Code

```bash
claude mcp add --scope user crypto-market -- node ~/mcp-servers/crypto-market-mcp/src/index.js
```

### 4. Проверь подключение

В Claude Code выполни `/mcp` — сервер `crypto-market` должен появиться в списке
подключённых, а инструменты — быть доступны в чате (например: "покажи топ-10 монет по капитализации").

---

## Вариант B: деплой в облако (для Cowork и claude.ai)

### 1. Задеплой на хостинг

Подойдёт любой хостинг с поддержкой Node.js и постоянно работающим процессом: Render, Railway, Fly.io и т.п. Ниже — пример для **Render** (есть бесплатный тариф):

1. Залей папку `crypto-market-mcp` в свой GitHub-репозиторий
2. На render.com: New → Web Service → подключи репозиторий
3. Настройки сервиса:
   - **Build Command**: `npm install`
   - **Start Command**: `npm run start:http`
   - **Environment Variables**: добавь `MCP_AUTH_TOKEN` со своим секретным значением (сгенерируй любую длинную случайную строку — это пароль для доступа к серверу)
4. Deploy. После сборки Render даст тебе публичный URL вида `https://crypto-market-mcp.onrender.com`

### 2. Проверь, что сервер отвечает

```bash
curl https://crypto-market-mcp.onrender.com/health
```

Должен вернуться `{"status":"ok",...}`.

### 3. Подключи как custom connector в Claude

В Settings → Connectors (это работает одинаково в claude.ai, Claude Desktop и Cowork):

1. "Add custom connector"
2. URL: `https://crypto-market-mcp.onrender.com/mcp`
3. В Advanced settings укажи заголовок авторизации — `Authorization: Bearer <твой MCP_AUTH_TOKEN>` (конкретное поле для этого зависит от текущего интерфейса Claude — если не найдёшь подходящее поле, посмотри актуальную документацию на support.claude.com, интерфейс мог измениться)
4. Подключи — сервер должен появиться в списке доступных инструментов

### 4. Важно про бесплатные тарифы хостингов

Бесплатные тарифы (например, у Render) обычно "усыпляют" сервис после периода бездействия — первый запрос после паузы может быть медленным (10-30 секунд на "пробуждение"). Для личного использования это обычно не критично, но об этом стоит знать.

### Безопасность токена

`MCP_AUTH_TOKEN` — это единственное, что защищает твой сервер от посторонних запросов, раз он публично доступен в интернете. Держи его в секрете, не публикуй в открытых репозиториях (используй переменные окружения хостинга, не хардкодь в коде).

## Ограничения бесплатных API

- CoinGecko free tier имеет лимит запросов (примерно 10-30 в минуту) — при частом использовании
  возможны кратковременные ошибки 429, это нормально, стоит просто повторить запрос.
- Yahoo Finance — неофициальный публичный эндпоинт, в теории может измениться без предупреждения.
- Если проект вырастет и лимиты станут проблемой, имеет смысл подключить CoinGecko Pro
  или Alpha Vantage с API-ключом — структура сервера уже это позволяет.

## Дальнейшее развитие

Этот сервер закрывает **общие рыночные данные** (цены, капитализация, макро-контекст).
DeFi-специфика (TVL, протоколы, yield) уже покрыта твоими существующими MCP —
DeFiLlama и Yield AgentKit, поэтому здесь она не дублируется.
